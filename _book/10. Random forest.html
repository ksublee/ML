10  Random forest

랜덤 포레스트 (random forest)는 bagged tree를 트리 간의 decorrelation을 통해 성능을 향상시킨 개념으로 bagging 앙상블 기법 중 하나이다.

  • 따라서 일반적인 bagging 보다 더 좋은 성능을 보일 여지가 높다.

일반적인 bagging에서 bootstrap을 통해 만들어진 훈련 데이터 셋들은 서로 공유하는 데이터가 많아 이를 통해 만들어진 tree들은 높은 상관관계를 지닌다.

  • 이에 따라 예측의 결과가 높은 variance를 유발한다.

랜덤 포레스트에서는 트리를 분할할 때, \(p\)개의 predictor중 임의로 선택된 \(m\)개의 predictor들을 후보로 하여 트리 분할을 진행한다.

  • 일반적으로 \(m \approx \sqrt{p}\)로 한다.

  • 즉, 랜덤 포레스트 방법은 트리 분할이 일어날 때 의도적으로 일부 predictor를 제외한다.

제외된 predictor들에는 strong predictor들도 있을 수 있는데 이들이 제외됨으로써 트리 간의 decorrelation이 발생한다.

이러한 제외 과정이 없는 bagging에서는 사실 트리들이 모두 비슷비슷하게 생겼을 가능성이 높다.

아래 그림은 bagging과 random forest 방법에서의 에러를 계산한 예제로, random forest의 좋은 성능을 보여주는 그림이다.

10.0.1 예제 : make_moons

초승달 모양 클러스터 두 개 형상의 데이터를 생성하는 make_moons를 활용한 예제

from sklearn.model_selection import train_test_split
from sklearn.datasets import make_moons

X, y = make_moons(n_samples=500, noise=0.30)
X_train, X_test, y_train, y_test = train_test_split(X, y)
import matplotlib.pyplot as plt
plt.scatter(X[y==1, 0], X[y==1, 1], s = 0.4)
plt.scatter(X[y==0, 0], X[y==0, 1], s = 0.4, color="red")

plt.show()

먼저 validation set 방법을 이용하여 tree와 bagging의 accuracy를 비교해 본다.

sklearn.metrics.accuracy_score는 실제값과 모형을 통한 예측값을 비교하여 예측이 얼마나 정확한지를 평가한다.

from sklearn.ensemble import BaggingClassifier
from sklearn.tree import DecisionTreeClassifier
from sklearn.metrics import accuracy_score

단독의 DecisionTreeClassifier를 사용한 예제를 먼저 살펴보자.

나중에 살펴볼 앙상블 방법보다는 더 적은 accuracy score를 보인다.

tree_clf = DecisionTreeClassifier(random_state=42)
tree_clf.fit(X_train, y_train)
y_pred_tree = tree_clf.predict(X_test)
print(accuracy_score(y_test, y_pred_tree))
0.896

아래 방법과 동일한 결과

tree_clf.score(X_test, y_test)
0.896

그 다음 적은 수(n_estimators=5)의 decision tree를 사용한 bagging이다.

  • 아래 코드에서는 max_samples를 전체 샘플 수보다 적은 100개로 지정하여 각 tree가 더 적은 샘플 수로 학습하여 다양한 모델이 되도록 유도하였다.
bag_clf = BaggingClassifier(
    DecisionTreeClassifier(), max_samples=100, n_estimators=5)

bag_clf.fit(X_train, y_train)

y_pred = bag_clf.predict(X_test)

print(accuracy_score(y_test, y_pred))
0.928

Decision tree의 개수인 n_estimators=300를 늘려보자.

적절한 n_estimators를 선택할 경우 accuracy score가 향상될 수 있다.

bag_clf = BaggingClassifier(

y_pred = bag_clf.predict(X_test)

print(accuracy_score(y_test, y_pred))

Decision boundary를 그려보자.

sklearn.inspection.DecisionBoundaryDisplay는 경계선을 그리기 위한 좋은 툴을 제공한다.

DecisionBoundaryDisplay.from_estimator의 첫번째 파라미터인 estimator에 대상이 되는 훈련된 classifier를 넣어 준다.

from sklearn.inspection import DecisionBoundaryDisplay

# subplots 함수를 사용하여 두 개의 axes를 가진 1x2 형태의 subplot을 생성

                        [tree_clf, bag_clf],

Random forest classifier의 feature_importances_는 각 feature 변수들의 중요도를 계산한다.

대략적으로 설명하자면 랜덤 포레스트에서는 각 트리별로 구한 importance를 feature 별로 평균내어 계산하는 개념이다.

중요도 값은 일반적으로 0에서 1 사이의 실수로 표현되며, 모든 특성의 중요도 값의 합은 1이다.

중요도 값이 클수록 해당 특성이 모델 예측에 더 큰 영향을 미치는 것으로 해석할 수 있다.

10.0.2.1 k-fold cross-validation error 추정해 보기

sklearn.model_selection.cross_val_score를 이용하여 편리하게 k-fold cross-validation 에러를 계산할 수 있다.

  • 여기서 'neg_mean_squared_error'는 MSE의 음수값을 나타내므로, mse 평균을 구할 때는 다시 마이너스를 취한다.

  • cross_val_score 함수의 기본 동작은 높은 값일수록 좋은 성능으로 간주하도록 되어 있어, 음의 평균 제곱 오차를 사용하도록 구현되어 있다.

X와 y 전체를 인자로 전달하여 에러를 계산한다.

from sklearn.model_selection import cross_val_score 
import numpy as np

mse_scores = cross_val_score(RandomForestRegressor(max_features='sqrt'), 
                             X, y, cv=5, scoring='neg_mean_squared_error')

mse_mean = - mse_scores.mean()

print("RMSE:", np.sqrt(mse_mean))
RMSE: 1038.8183143661859

단순 트리 모형과 비교

from sklearn.tree import DecisionTreeRegressor

tr = DecisionTreeRegressor()

tr_mse_scores = cross_val_score(tr, X, y, cv=5, scoring='neg_mean_squared_error')
tr_mse_mean = - tr_mse_scores.mean()

print("RMSE:", np.sqrt(tr_mse_mean))
RMSE: 1247.2251655186146

Bagging과 비교

from sklearn.ensemble import BaggingRegressor

bc = BaggingRegressor(estimator=DecisionTreeRegressor())

bc_mse_scores = cross_val_score(bc, X, y, cv=5, scoring='neg_mean_squared_error')
bc_mse_mean = - bc_mse_scores.mean()

print("RMSE:", np.sqrt(bc_mse_mean))
