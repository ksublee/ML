12  Boosting

12.1 개요

Boosting은 약한 학습기(weak learner)들을 결합하여 강한 학습기(strong learner)를 만드는 ensemble 방법론 중 하나이다.

Boosting은 처음에는 각각의 데이터 포인트에 대해 동일한 가중치를 부여한 상태에서 학습을 시작한다.

  • 즉, 일반적인 머신 러닝 방법을 적용한다.

이후, 이전에 학습된 모델이 잘못 예측한 데이터에 대해 더 높은 가중치를 부여하여, 다시 새로운 모델을 학습한다.

  • 이전 학습기의 오분류된 샘플에 주목하고, 다음 학습기가 이를 보완하여 전체 모델의 성능을 향상하고자 하는 것

  • 오분류에 주목하기 위해 개별 학습기는 약한 학습기를 사용

이를 반복하여 여러 개의 약한 학습기를 결합(투표, 평균, 더하기)하는 방식을 취한다.

각각의 약한 학습기들이 모여서 궁극적으로 높은 성능을 보인다.

부스팅 (boosting)은 bagging과 마찬가지로 트리 모델만이 아닌 다른 일반적인 통계적 학습에도 적용할 수 있다.

12.2 Adaboost

에이다부스트 (adaboost)는 adaptive boosting의 줄임말로 인기있는 부스팅 방법 중 하나이다.

트리뿐만 아니라 다른 예측기들을 통해서도 adaboost를 적용할 수 있다.

여러 약한 학습기, 예를 들어 분기가 하나인 트리를 순차적으로 학습하여 최종적으로 투표 혹은 평균내는 과정으로 강한 학습기를 만든다.

분류기의 경우 다음과 같은 방식으로 adaboost는 작동한다.

  • 첫 번째 분류기를 훈련세트에서 훈련시키고 예측한다.

  • 잘못 분류된 샘플의 가중치를 높인다.

  • 두 번째 분류기는 업데이트된 가중치를 지닌 샘플을 사용하여 훈련하고 다시 예측한다.

  • 또다시 잘못 분류된 샘플의 가중치를 업데이트한다.

12.2.1 훈련

자세한 훈련 메카니즘은 구현된 알고리즘에 따라 조금씩 다르다. 아래에 소개된 내용은 기본적인 형태의 AdaBoost 분류 알고리즘에 대한 것이다.

12.2.1.1 샘플의 가중치

먼저, \(i\)-번째 샘플의 가중치 \(w^{(i)}\)는 초기에는 \(\frac{1}{m}\)으로 초기화한다. 여기서 \(m\)은 데이터의 수.

첫 번째 예측기(약분류기, weak learner)가 학습되고, 가중치가 반영된 에러율 \(r_1\)을 계산한다.

각 차례 \(j\)에서 가중치 적용 에러율은 다음과 같다.

\[ r_j = \sum_{i=1}^{m} w^{(i)} \cdot \mathbb{1}_{ \{\hat y_j^{(i)} \neq y^{(i)} \} } \]

  • \(\hat y_j^{(i)}\)는 \(j\)번째 예측기가 \(i\)번째 샘플에 대해 예측한 값

  • \(y^{(i)}\)는 실제 레이블 즉, \(r_j\)는 \(j\)번째 예측기가 잘못 분류한 샘플들의 가중치 합을 의미한다.

  • \(r_j\)가 0에 가까울수록 좋은 예측기

    • 단, \(r_j > 0.5\)이면 해당 학습기는 성능이 매우 낮다는 것이므로 제외할 수 있음.

12.2.1.2 예측기의 가중치

AdaBoost에서는 알고리즘에 따라 예측기에도 가중치가 적용되는 경우도 있다.

각 \(j\)번째 예측기(learner)에 부여되는 가중치 \(\alpha_j\)는 다음과 같다.

\[ \alpha_j = \eta \log \frac{1 - r_j}{r_j} \]

  • \(r_j\)가 0에 가까울수록, 즉, 예측기가 정확할수록 \(\alpha_j\)의 값이 크다.

  • 예측기가 부정확하다면 \(\alpha_j\)의 값은 0보다 작을 수도 있다.

  • 여기서 \(\eta\)는 학습률 하이퍼파라미터로 기본값은 1이다.

12.2.1.3 가중치 업데이트

\(j\)번째 예측기의 학습이 끝난 후, 각 샘플의 가중치는 다음 규칙에 따라 업데이트된다.

\[ w^{(i)} \leftarrow \left\{\begin{array}{ll} w^{(i)}, & \text{if } \hat y_j^{(i)} = y^{(i)} \text{ (분류 성공) }\\ w^{(i)} \exp(\alpha_j) , & \text{if } \hat y_j^{(i)} \neq y^{(i)} \text{ (분류 실패) } \end{array}\right. \]

  • 업데이트 된 샘플의 가중치는 다시 \(\sum_{i=1}^{m} w^{(i)}\)로 나누어 정규화한다.

이 과정의 핵심은 다음과 같다.

  • 분류에 실패한 샘플의 가중치는 증가한다.
  • 특히 성능이 좋은 예측기(\(\alpha_j\)가 큼)가 실패한 샘플일수록 가중치 증가폭이 크다.
  • 결과적으로, 이후 학습 단계에서는 이전 단계에서 잘 맞추지 못했던 샘플에 더 많은 관심을 두게 된다.

이 과정을 \(j = 1, 2, \dots\)에 대해 반복하며 학습을 진행한다. 일반적으로는

  • 사전에 지정된 예측기 개수에 도달하거나,
  • 에러율이 0인 완벽한 예측기가 등장하면 학습을 종료한다.

12.2.2 예측

훈련이 완료된 booster에 새로운 \(x\)값을 대입하면 booster 내의 각 예측기들이 해당 \(x\)에 대한 \(y\)를 예측하여 클래스 \(\hat y_j(x)\)를 반환할 것이다.

  • 예측기 별로 같은 클래스를 예측할 수도 있고 다른 클래스로 예측할 수도 있다.

  • 일반적인 상황이라면 다수결로 클래스를 정하면 된다.

그런데, AdaBoost에서는 각 예측기들의 가중치 (중요도)가 다르므로 이를 감안하여 가중 다수결을 통해 예측 결과를 종합한다.

즉, 예측을 할 때는 가중치 \(\alpha_j\)들의 합을 최대화 하는 클래스가 예측 결과가 된다.

\[ \hat y (x) = \arg\max_{k} \sum_{j = 1}^N \alpha_j \cdot \mathbb{1}_{\hat y_j (x) = k} \]

즉, 다수결 투표와 비슷하지만 분류기의 성능에 따라 투표 가중치 \(\alpha_j\)를 달리준다. (\(\alpha_j =1\)이면 일인 일투표권과 같음)

12.2.2.1 sklearn.ensemble.AdaBoostClassifier

Scikit-learn에서 구현된 Adaboost.

Scikit-learn에서는 SAMME혹은 SAMME.R이라는 조금 변형된 알고리즘을 사용하며, 다중(\(K\)) 클래스 문제에서도 사용할 수 있다.

이 알고리즘에서 \(j\)번째 예측기 가중치를 구하는 식은

\[ \alpha_j = \eta \left( \log \frac{1 - r_j}{r_j} + \log (K-1) \right) \]

이다. 많다.

12.3 Gradient boosting

또하나의 인기있는 부스팅은 gradient boosting이다.

Gradient boosting 또한 여러 개의 약한 예측 모델(weak learner)을 조합하여 강한 예측 모델(strong learner)을 만드는 앙상블 학습 방법이다.

Gradient boosting의 경우 부스팅은 데이터가 아닌 residual(잔차) 혹은 손실 함수의 기울기(gradient)를 이용하여 학습하는 개념이다.

  • 즉, residual \(= y_i - \hat y_i = y_i - \hat f(x_i)\)이 점차 작아지도록 학습

  • residual은 모형의 예측과 관측값의 차이라는 점에서, 모형의 예측이 잘 들어맞지 않는 부분을 중점적으로 다시 학습하는 방식이다.

  • 공부를 할 때 틀린 문제들을 위주로 다시 학습하는 것과 비슷한 원리

12.3.1 알고리즘

12.3.1.1 표기법

다음 알고리즘을 살펴보기 전에 다음의 notation을 알아두자.

  • $ f_t $ : \(t\) 단계에서 적합에 사용되는 작은 트리 (약한 학습기)
  • $ f^{(t)}$ : 1부터 \(t\) 단계까지 생성된 작은 트리들을 모두 합한 부스팅된 모형.
    • 정확히는 학습률(shrinkage parameter)을 곱하여 더함

\[ \hat f^{(t)}(x) = \sum_{s=1}^{t} \lambda f_s(x) \]

12.3.1.2 회귀문제에서의 알고리즘

Regression 모형에서 gradient boosting은 다음의 알고리즘을 따른다.

  • \(\hat f^{(0)}(x) = 0, \quad r_i^{(0)} = y_i\)로 한다.

  • \(t = 1, 2, \cdots, T\)에 대해 다음을 반복.

    • 작은 트리 \(\hat f_t\)를 훈련데이터 \((X, r^{(t-1)})\)에 적합. 즉, \(\hat f_t(x_i) \approx r^{(t-1)}_i\)

    • \(\hat f\)를 업데이트 한다. 이때 shrinkage parameter \(\lambda\) (학습률)를 이용한다.

      • $f^{(t)}(x) f^{(t-1)}(x) + f_t(x) $

      • \(t\)-step에서의 residual을 업데이트 한다. \(r_i^{(t)} \leftarrow r_i^{(t-1)} - \lambda \hat f_t(x_i)\)

        • 이렇게 하면 \(r_i^{(t)} = y_i - \sum_{s=1}^{t} \lambda \hat f_s(x_i) = y_i - \hat f^{(t)}(x_i)\)가 되어 residual의 정의와 일치한다.

        • \(\sum_{s=1}^{t} \lambda \hat f_s(x\_i)\)는 \(t\) 단계까지 형성된 트리를 합한 것으로 \(t\)단계까지 부스팅된 모형

  • 최종 모형은 다음과 같다

\[\hat f^{(T)} (x) = \sum_{t=1}^{T} \lambda \hat f_t(x) \]

분류(classification) 문제에서도 Gradient Boosting을 유사한 방식으로 적용할 수 있다.

다만, 이 경우에는 residual 대신 손실 함수의 음의 기울기(negative gradient)를 새로운 목표값으로 사용한다.

12.3.4 왜 gradient boosting인가?

Gradient boosting은

손실 함수를 최소화하는 방향(음의 gradient)으로 예측 함수 자체를 점진적으로 업데이트하는 방법이다.

각 단계에서 새로운 weak learner는

  • 이전 단계 모델이 가장 잘못 예측한 방향,
  • 즉 손실 함수의 음의 gradient를 근사하도록 학습된다.

12.3.4.1 제곱오차 손실에서의 해석

회귀문제에서 손실 함수를

\[ \ell(y, \hat y) = \frac{1}{2} (y - \hat y)^2 \]

라고 하자.

이 loss function의 \(\hat y\)에 대한 미분(gradient)을 구하여 \(\hat y = \hat f^{(t-1)}(x_i)\)에서 계산하면,

\[ \left. \frac{\partial \ell(y_i, \hat y)}{\partial \hat y} \right|_{\hat y = \hat f^{(t-1)}(x_i)} = \left. \frac{1}{2} \frac{\partial (y - \hat y)^2}{\partial \hat y} \right|_{\hat y = \hat f^{(t-1)}(x_i)} = - (y_i - \hat f^{(t-1)}(x_i)) = - r_i\]

즉, residual은 손실 함수의 음의 gradient와 정확히 일치한다.

12.3.4.2 Gradient boosting 업데이트 구조

Gradient boosting의 업데이트 과정을 다시 살펴보면

\[ \hat f^{(t)}(x_i) = \hat f^{(t-1)}(x_i) + \lambda \hat f_t(x_i) \]

그런데 \(\hat f_t\)는 residual \(r^{(t-1)}\)에 fitting 되는 것(근사하는 것)이므로, \(r^{(t-1)}_i \approx \hat f_t(x_i)\)라고 할 수 있다.

\[ \hat f^{(t)}(x_i) \approx \hat f^{(t-1)}(x_i) + \lambda r^{(t-1)}_i \]

따라서, \[ \hat f^{(t)}(x) \approx \hat f^{(t-1)}(x) - \lambda \left. \frac{\partial \ell(y_i, \hat y)}{\partial \hat y} \right|_{\hat y = \hat f^{(t-1)}(x)}\]

즉, gradient boosting은 손실함수가 감소하는 방향(음의 gradient)으로 앙상블 모형 \(\hat f^{(t)}\)을 업데이트 하는 것이다.

(\(r^{(t-1)}\)에 가장 잘 적합되는 weak learner \(\hat f_t\)를 찾는 것은 손실함수의 음의 gradient에 가장 잘 적합하는 \(\hat f_t\)를 찾는 것과 동치)

  • 이는 최적화 문제에서 음의 gradient 방향으로 파라미터 공간에서 파라미터 값을 계속 이동하여 어떤 비용함수 \(g\)를 최소화하는 점을 찾는 것과 동일한 메카니즘

\[\hat \theta_{t} = \hat \theta_{t-1} - \lambda \nabla g(\theta)|_{\theta = \hat\theta_{t-1}} \]

  • 이런 관점에서 gradient boosting은 함수 공간에서의 경사하강법이라고도 부름.

12.3.4.3 임의의 loss function에 대해서도 적용 가능

이 아이디어를 확장하면 편차 제곱 loss function 뿐만 아니라 임의의 loss function에 대해서도 gradient boosting 방법을 적용할 수 있다.

함수 \(f^{(t)}\)가 \(t\)단계 모델일 때, 경사하강법 원리에 따라, 다음과 같이 업데이트를 해야 할 것이다.

\[ f^{(t)}(x) = f^{(t-1)}(x) - \lambda \left. \frac{\partial \ell(y_i, \hat y)}{\partial \hat y} \right|_{\hat y = \hat f_i^{(t-1)}(x)} \]

  • 함수공간에서의 경사하강법

임의의 loss function \(\ell(y, \hat y)\)에 대해, \(t\) step에서 weak-learner를 적합할 pseudo-residual을 각 데이터 \(i\)에 대해 다음과 같이 정의하면,

\[ r^{(t-1)}_i = - \left. \frac{\partial \ell(y_i, \hat y)}{\partial \hat y} \right|_{\hat y = \hat f_i^{(t-1)}(x)} \]

이고, \(\hat f_t(x_i) \approx r^{(t-1)}_i\)이므로, 위 업데이트 룰은 다음과 같이 근사된다.

\[ f^{(t)}(x) \approx f^{(t-1)}(x) + \lambda \hat f_t(x) \]

12.3.5 Gradient Boosting의 주요 구성 요소

일반적인 Gradient Boosting은 다음의 세 가지 구성 요소로 이루어진다.

  • 손실 함수 (Loss function) 모델의 예측이 실제값과 얼마나 차이가 나는지를 측정하는 기준이다. Gradient boosting에서는 손실 함수의 음의 gradient가 각 단계에서 학습의 목표가 된다.

  • Weak learner 각 단계에서 손실 함수의 음의 gradient(또는 residual)를 근사하는 약한 예측기이다. 보통은 얕은 의사결정트리가 사용되지만, 다른 회귀 모델도 사용할 수 있다.

  • Additive model Weak learner를 하나씩 순차적으로 추가하는 가법적 모델이다. 각 단계는 손실 함수를 감소시키는 방향으로 진행되며, 학습은 정해진 반복 횟수에 도달하거나 성능개선이 멈추면 종료된다.

12.3.6 분류 문제 gradient boosting

분류 문제의 gradient boosting 또한 회귀 문제의 gradient boosting과 매우 흡사하다.

회귀 문제에서처럼 손실 함수를 정의하고, 손실 함수의 \(\hat y_i\)에서의 미분을 통해 그것의 음의 gradient(미분)를 계산한다.

12.3.6.1 이진분류 문제

먼저 이진분류 문제를 생각해 보자.

만약 손실 함수를 binary cross entropy인

\[ - y_i \log(\hat p_i) - (1 - y_i) \log(1 - \hat p_i), \quad \hat p_i = \frac{1}{1 + e^{-\hat y_i}}\]

로 정의해 보자. 여기서 \(\hat y_i\)는 모델이 출력하는 실수값 (logit)이며, \(\hat p_i\)는 이에 대한 sigmoid 변환(예측확류)이다.

이 손실 함수를 \(\hat y_i\)에 대해 미분한 뒤 음의 gradient를 계산하면, pseudo-residual은 다음과 같이 단순화된다.

\[ y_i - \hat p_i\]

  • 단, 이전에 설명한 것처럼 gradient boosting은다른의 loss function에 대서도해 적용할 수 있으며, \(\hat y\)에 대한 음의 미분을 residual로 삼을 수 있다.

주의할 점은 분류 문제라고 할지라도 실제로 훈련되는 트리는 실수값으로 적합되는 regression tree이다. (residual이 실수값이므로)

따라서, 트리의 예측값은 실수이므로, 예측의 과정에서는 아래의 sigmoid 함수를 이용하여, 앙상블 모형의 예측값인 \(\hat y_i\)를 확률값으로 변환하고, 더 높은 확률을 가지는 클래스로 예측한다.

\[ \hat p_i = \frac{1}{1 + \exp(-\hat y_i)}, \quad 1- \hat p_i = \frac{\exp(-\hat y_i)}{1 + \exp(-\hat y_i)}\]

(즉, 이진 분류 문제의 경우 \(\hat p_i > 0.5\)이면 1번 클래스로 아니면 0번 클래스로 분류됨)

12.3.6.2 다중 클래스 문제

Multi-class의 경우, sklearn에서는 각 boosting 단계에서 \(K\)개(class의 수)의 tree들을 생성한다.

즉, 각 트리는 특정 \(k\) 클래스에 속하는지 않는지를 판단하는 트리이며, 이 방식을 One-vs-Rest 전략이라고 한다.

  • 각 클래스마다 하나의 이진 분류 문제를 만들어 해당 클래스에 속하는지 여부를 판단

예측의 단계에서 각 트리의 앙상블에서 계산된 값은 softmax 함수를 통하여 각 클래스에 속할 확률로 변환된다.

\[ \frac{e^{\hat y_k}}{\sum_{k=1}^K e^{\hat y_k}} \]

Scikit-learn의 multi-class에 대한 gradient boost에서 모델의 초기값으로는 각 클래스 \(k\)에 대해 \(\log(\mathbb P(Y=k))\)에 대한 추정치를 이용한다.

  • 즉, 샘플 비율을 사용하여, 훈련 샘플의 수 중 \(k\) 클래스에 속하는 비율에 로그를 취한 값

  • 각 스텝마다 트리가 총 클래스의 수, \(K\)개만큼 필요하므로, 초기값 또한 클래스 별로 \(K\)개 필요하다.

Binary 문제에서 log odds를 초기값으로 썼던 것과는 조금 다르나, 충분히 많은 estimator들을 사용할 경우 초기값은 크게 중요하지는 않다.

.estimators_는 리스트의 리스트로 구성되어 있다.

  • 리스트의 첫번째 인덱스는 boosting 단계를 나타내고, 두번째 인덱스는 각 스텝에서 등장하는 트리들의 번호(클래스 번호에 대응)이다.

아래 코드에서는 estimators_에 저장된 트리들을 이용하여 bootstrap 앙상블 결과를 재현한다.

트리 모형의 예측치들을 합산할 때, gb_clf.learning_rate을 곱해준다.

y_hat_t = np.zeros(len(gb_clf.classes_))

for i in range(len(gb_clf.classes_)):
    y_hat_t[i] = np.log(len(y_train[y_train == i]) / len(y_train)) 


for i in range(gb_clf.n_estimators_):

    tree_prediction = np.zeros(3)
    
    for j in range(3):
        
        tree_prediction[j] = gb_clf.estimators_[i][j].predict(X_test[:1])
                
    y_hat_t += gb_clf.learning_rate * tree_prediction
    
    print("Raw score by sum of tree predictions : ", y_hat_t)
    print("Probability by sum of tree predictions : ", np.exp(y_hat_t) / np.sum(np.exp(y_hat_t)))
